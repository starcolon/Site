What is Functional Programming?

Developers treat object as a first-class citizen in Object-Oriented Programming. Whereas in Functional Programming, functions are considered first-class citizen.

In comparison with a famous programming paradigm, OOP, here are conceptually notable differences:

  • Functions have no state (pure function). Same inputs always produce the same outputs. http://en.wikipedia.org/wiki/Pure_function
  • Functions are first-class so we can pass functions as their arguments. http://en.wikipedia.org/wiki/First-class_function
  • Write functions in a sequence of instructions on what to do rather than using states. http://en.wikipedia.org/wiki/Monad_(functional_programming)

Should you always write Functional programming style?

Functional programming is not a silver bullet to every possible problems.
In some tasks, writing your code in Functional style complicate it further in some degree. Functional code can be thoughtfully written in a clearer appearance but not always. But when you tend to write code based on Functional philosophy, try your best to tidy it for other people who will look at your code later (even your future self).

Lots of people dislike the way Functional code looks so let's change their minds.

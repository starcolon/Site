C++

C++ was popular and it still is. Beginners would probably find memory allocation, clearance, and pointers a bit brain-hurting. Yet, after master all of this fundamental knowledge, the next brick wall they would hit hard are:

const and pointer ★★★☆☆

You don't need to be a C++ uber to know the meaning behind the following function signature: 

						
							int* doSomething(DATA* a,long* b);

But once const involves, it will look less friendly to beginners. 

						
							int* doSomething(const DATA* a,long* b);

Beginners sometimes just realise they suffer from seizure and migraine when they see const in different positions. 

						
							int* doSomething(const DATA* a,long* b) const;
							int* doSomething(const DATA* a,long const* b);
							const int* doSomething(const DATA*,long const* b);

							// Oh...

const with passing by ref ★★★★★

Passing arguments by reference when associates with const is a complete onslaught against beginners, literally... 

						
							const int* doSomething(const DATA* &a, long const* &b);

							// Beginners shut the lid, go burn their c++ for dummy books

Proper uses of passing by ref along with const makes function handles memory very efficiently. But this may take some time for beginners to understand and accustom themselves to use it.

Python

The grammar and syntaxes of Python is easy for everybody to learn and gain familiarity quickly. But the awaiting brick wall are:

Decorator ★★★☆☆

You need to understand nested functions and closures to master it.

First off, when seeing the decorated function as follows. 

						
							@plus_one
							def f(a):
							  return a*10

							# Where the function plus_one looks like:
							def plus_one(g):
							  def _(n):
							    return n+1
							  return _

People who just learn Python would probably wonder what the hell @ sign decorator does with function f(a). To interpret decorator into basic functions, they will look like following. 

						
							def f(a):
								return a*10

							# Here f() is decorated (modified)...
							f = plus_one(f)

It's actually easy. But for beginners, it may probably take some time to remove confusion and finally they will realise how simple (and beautiful) it is.

Ruby

Ruby is clearly one of the frienliest programming languages by design. However, apart from its simplicity, there is still something a bit hard to learn at the beginning.

Block ★★☆☆☆

						
							result = doSomething "foo" do |bar|
								puts bar
							end

For those who just land on first lessons of Ruby would frown at it. Why doSomething() returns result and passes bar to another block at the same time?

This works much like a callback in general sense. Ruby just makes it more verbose and less risky of introducing pyramid of doom.

Functions with ? or ! ★★★☆☆

Mindblown moment for people from different languages. At the first glance, functions with ? or ! likely lead some traffics to StackOverflow.com. 

						
							model.save!
							if model.save? 
								...
								...
							end

							# Nah....

JavaScript

JavaScript is relatively more flexible than other scripting languages yet it supports wide range of programming paradigms. Learning JavaScript from the fundamental level is comfortable until the following.

For loops ★★★☆☆

JavaScript comes with a few versions of for loops. 

						
							var v = ['a','b','c'];

							for (var i in v)
								console.log(this,v[i]);

							for (var n of v)
								console.log(this,n);

							v.forEach((n,i) => console.log(this,n));

Three loops above work differently and scope the variable this even more differently (In reality, this often causes mistakes). Beginners may find it complicated at first glance.

Coercion to boolean ★★★★★

Let's see how following expressions evaluate to boolean. 

						
							![] // falsy

							![1] // falsy

							!([] + []) // truthy ... heck??

							!!{foo: 'bar'} // truthy

							!!{} // still truthy ..Hmmm

							!!-1 // truthy

							!'' // truthy

							!' ' // one space becomes falsy 

							s = new Set();
							!!s // Empty set is still truthy

A complete intuition of boolean coercion in JavaScript still remains controversial, not just only beginners. On StackOverflow you may find a number of questions playing around with different combinations of these.

Haskell

When compare to other mainstream languages in the industry, Haskell looks much like an alien's language invented by a counterpart of God who had a strong desire to destroy all mandkind.

List pattern capture ★★★★☆

For beginners, writing a function definition which matches all possible patterns is much more difficult than capturing a legendary pokémon. See example below: 

						
							flip_every_two :: [a] -> [a]
							flip_every_two [] = []
							flip_every_two n:[] = [n]
							flip_every_two n:ns:[] = [head ns,n]++flip_every_two $ tail ns

For beginners, it's hard to come up with list patterns like n:[] or n:ns, etc.

Monad ★★★★★

None of articles, tutorials, books, or any Haskell experts say Monad is easy. If they do they're drunk. 

						
							[0..100] >>= (\n -> [x-1,x,x+1])

Or more general use of bindings 

						
							filterM pred xs = do
  							bools <- mapM pred xs
  							return [ x | (x,True) <- zip xs bools ]

  						-- Excerpt from Facebook Haxl

Monad is a set of simple concepts, but visually complicated when comes into designing. Monoid is also a related area of Haskell's common feature which involves some difficulty to learn for the beginner.

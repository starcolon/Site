Self-Balancing Trees

Image courtesy of: http://en.wikipedia.org/wiki/Self-balancing_binary_search_tree

Self-balancing tree is a binary search tree data structure which has an ability to rebalance its branches so the node access is much more faster than simple binary trees.

In general, self-balancing tree warrants complexity of O(log(N)) for worst case scenarios whereas simple binary-search tree can make it only O(N).
Extra functionality of self-balancing mechanisms of self-balancing trees can conveniently be implemented on top of the general binary search tree. Thus, we can start off with building a simple binary search tree data structure and reuse it for latter implementations.

Making Binary Search Tree in JavaScript

As simple as it looks, a binary tree is constructed from a bunch of connected nodes where each node is allowed to have up to 2 children. The top-most node does not belong to any parent nodes is called root.
Thus, we can consider making a smallest particle of a binary tree which as these properties:
  • Its value (perhaps, key-value pair)
  • Left children branch (sub tree)
  • Right children branch (sub tree)

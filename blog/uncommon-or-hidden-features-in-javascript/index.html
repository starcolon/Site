Some tricks with JS

When referring to JavaScript, I'm talking about ECMAScript 6

Since born, JavaScript has introduced a series of revisions and now it meets (or is approaching) ECMAScript 6's specifications which brings lots of new fancy functionalities and syntaxes. This article won't overly focus on new features come with ECMAScript 6 but it's better to be aware of.

Named-argument-like just like Python

Using variable assignment statements as named-argument-like code may help inspecting the argument a bit easier. 
					
						// Call it with named arguments
						doJob(
							taskId=10001,
							isHaltWhenFail=false,
							isSynchronous=false,
							params=['a','b']
						);
No matter how long the argument list is - this simple trick could implicitly add some semantic to those arguments for the sake of simplicity.
CAVEAT This trick potentially assigns the values to the named variable upon a function call. In case the variable names are duplicate with the previously defined ones, their values will be replaced.

Unsigned bitwise shift

Everyone knows bitwise shift operators << and >> which are signed operators. Nonetheless, there available another unsigned bitwise operator in JavaScript. 
					
						// Signed bitwise shifts
						16 >> 1; // 8
						-16 >> 1; // -8

						// Unsigned bitwise shifts
						16 >>> 1; // 8
						-16 >>> 1; // 2147483640 (Signed bit also shifted)

Replace If statement with logical and

You can shrink the following simple If statement. 
					
						if (this || that){
							this.a = that;
							done();
						}
Into this 
					
						// One liner
						(this || that) && ((this.a = that) || done());
The technique is simple. It short-circuits the logical and statement whenever the term is not evaluated to true. Applying this technique to a bit more complicated nested statement like following. 
					
						for (var a of list){
							if (a.isDone){
								a.val++;
							}
							if (a.isReady && a.val>42){
								do(a)
							}
						}
So it becomes neater and less pyramid-of-doom look. 
					
						for (var a of list){
							a.isDone && a.val++;
							a.isReady && a.val>42 && do(a);
						}

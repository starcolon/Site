EX1: Generate a list of tuples (1,2),(3,4),(5,6),...

JS

JavaScript is very handy at this job. Go for ES6 generator for lazy evaluation. 
					
						function* gen(n){
							yield [n,n+1];
							yield* gen(n+2);
						}

						var tuples = gen(1); // Initiate
						// Go...
						tuples.next().value; // [1,2]
						tuples.next().value; // [3,4]
						tuples.next().value; // [5,6]

Python

Let's make an infinite generator for tuple in a similar way.
				
					def gen(n):
						while True:
							yield (n,n+1)
							n = n + 1

					# Initiate
					tuples = gen(1)

					# Go go!
					next(tuples) # (1,2)
					next(tuples) # (3,4)
					next(tuples) # (5,6)

Haskell

People say dammit. It's too easy. 
				
					-- Infinite run
					[(i,i+1) | i <- [1,3..]] 

					-- [(1,2),(3,4),(5,6).......... until you beg it to stop.

EX2: Fibonacci tuples (1,1),(2,3),(5,8),...

JS

Generator comes into play yet again. 
					
						function* gen(a,b){
							yield [a, b];
							yield* gen(a+b, a+2*b);
						}

						var tuples = gen(1,1); // Initiate first 2 elements
						// Go...
						tuples.next().value; // [1,1]
						tuples.next().value; // [2,3]
						tuples.next().value; // [5,8]

Python

Similar formula with the preceding JS snippet. 
				
					def gen(a,b):
						while True:
							yield (a,b)
							a, b = a+b, a+2*b

					# Initiate
					tuples = gen(1,1)

					# Go go!
					next(tuples) # (1,1)
					next(tuples) # (2,3)
					next(tuples) # (5,8)

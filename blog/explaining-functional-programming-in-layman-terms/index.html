What are Functional Programming,
Monad, Monoid, Applicative, Functor ??
5 June 2017 | Tao P.R.

Prelude:*

If you come across this article, you may know Functional programming already. Now I'll gently walk you through the world of it and gain familiarity with some of its principal elements.

What is Functional Programming ?

The rough idea is always thinking of a logic in your program as a function which takes an input, and produce another piece of the data, an output.

In an opposite side of OOP, Functional programming tends to see every piece of your program as functions, rather than objects. Functions never keep any internal state nor care the outside world. They just operate the logic on the data to produce another output. That's it.

Characteristics of Functional Programming

  • Functions never draw effect on outer scope. They just take inputs and produce output. (purity of function)
  • Given the same inputs, no matter when, calling the same function will always yield the same outputs.
  • A function can be passed as an argument of an another function. (Function as a first-class citizen)

[1] Functions

Simple. A function is a mapping from one data to another data.
Data type may or may not be retained.

`f: a -> b`

As aforementioned, function `F` doesn't change the input value of type `a` after execution, it just produces an output of type `b`.

[2] Algebraic Data Types

For simpler notation, Algebraic data types are:

  • A categorical data. Think of enum of Colors which the value can be one of red, yellow, etc. So the datatype can be expressed as 
                
              data Color c = Red | Yellow | White | Custom c
    A variable can represent one single category at a time. It can't be red and yellow simulteneously. We call this behaviour an Addition. So this data type is called Sum type.

  • A composition of values. Think of tuples, struct, etc. where one variable can hold multiple attributes at a time. 
                
            data NameAndAge name age = NameAndAge name age
            
            -- So we can have variables like so
            let a = NameAndAge "Peter" 20
            let b = NameAndAge "Joe" 31
    We call this behaviour a Multiplication. So this data type is called Product type.

In Haskell, one of the famous algebraic types is called Maybe which holds a nullable value. 

          
            data Maybe a = Nothing | Just a

            let x = Just 10 -- valued
            let y = Nothing -- or null
            let z = if blahblah then Nothing else Just "work"

[3] Functors

Functors are algebraic type which you can "map" the value inside into an output. Functor will automatically wrap the output as a new Functor.

Example, List is a functor. Maybe is also a functor.

So we can "map" a list into another list. 

            
              let v = [1,2,3,4] 

              -- map into another list
              -- by multiplying each element with 10
              fmap (*10) (v) -- [10,20,30,40]

Also, we can "map" a Maybe into another Maybe. 

            
              -- map into another Maybe
              -- by multiplying the value with 10
              fmap (*10) (Just 10) -- Just 100
              fmap (*10) (Nothing) -- Nothing

As above, the output is wrapped automatically into a new Functor.

[4] Applicative

Applicative is a more advanced Functor.
It also accepts the mapping function which is wrapped as another Applicative!

Example, List is an Applicative.
So you can apply a List of function to produce a new list. 

            
            let v = [0, pi/2, pi]
            let f = [sin, cos]

            -- Apply list of functions onto a list

            f <*> v

            -- result : [sin(0), sin(pi/2), sin(pi), cos(0), cos(pi/2), cos(pi)]

Maybe is also an Applicative.
Likewise, you therefore can apply a Maybe of function to create a new Maybe value. 

            
            let m = Just pi
            let f = Just sin

            -- Apply

            f <*> m -- result : Just 0

            -- What if Nothing:

            f <*> Nothing -- result : Nothing

Applicative operator <*> is like a cross product of two operands.

[5] Monad

Monad is an Applicative which accepts a function which takes the value and produce a new Monad of the output, aka. flatMap.

Again, List is a Monad! Thus, if you have a List of Int, you can apply a function which takes an Int and produces another List of Int ... yes, it's definitely a FlatMap. 

            
            let v   = [1,2,3]
            let f x = [x, x*10]

            -- flatMap!

            v >>= f

            -- Result : [1,10,2,20,3,30]

            -- Or chain even further!

            v >>= f >>= f

            -- Result : [1,10,10,100,2,20,20,200,3,30,30,300]

            let g x = if x<10 then [] else [x+1]
            v >>= f >>= g

            -- Result : [11,21,31]

The operator >>= of Monad is widely known as a flatMap.

Yet another example, Maybe is also a Monad.
So you can flatMap it and create another Maybe! 

            
            let f n = if n<5 then Nothing else Just (n+1)

            -- Try out on some cases

            (Just 5) >>= f -- Get Just 6

            (Just 9) >>= f -- Get Just 10

            (Just 0) >>= f -- Get Nothing

Functor vs Applicative vs Monad ?

Let's wrap up again and highlight the differences among them.

  Key Operator Operand Function Operator Output
Functor map `f: a -> b` Functor of `b`
Applicative <*> Applicative of `f: a -> b` Applicative of `b`
Monad >>= `f: a ->` Monad `b` Monad of `b`

Example with List!

  Key Operator Operand Function​ (Sample) Operator Output
List[Int] map f(a: Int): Int List[Int]
List[Int] <*> List[ f(a: Int): Int ] List[Int]
List[Int] >>= f(a: Int): List[Int] List[Int]

Another example with Maybe!

  Key Operator Operand Function​ (Sample) Operator Output
Maybe[Int] map f(a: Int): Int Maybe[Int]
Maybe[Int] <*> Maybe[ f(a: Int): Int ] Maybe[Int]
Maybe[Int] >>= f(a: Int): Maybe[Int] Maybe[Int]

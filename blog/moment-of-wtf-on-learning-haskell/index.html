Prelude

If you struggle learning Haskell as a Pythonista or C++ nerds, or even other languages with imperative background, you're definitely not the only one. Functional programming is a different paradigm which can apply to other modern languages like Ruby, Python, JS, etc. But learning to write pure functional code in Haskell is totally a different topic. Developers with a strong mindset trained for imperative or procedural programmings may find it noticeably hard to adopt different thinking for Haskell. Knowing lots of imperative languages doesn't seem to take advantage.

Passing negative number

You may do this when you want to pass a negative number to a function in Python. 
				
					def f(a,b):
						return a*b

					f(5,3) # Just works
					f(-5,3) # Just works
But it hurts you when you try to do the same thing in Haskell. 
				
					let f a b = a*b

					f 5 3 -- works
					f -5 3 -- WTF! Non type-variable argument in the constraint: Num (a -> a -> a)
Haskell sees -5 as an operation "minus something by five", not a number. This would blow your mind. Function first class doesn't always magically make life easier, it somehow hurts at the first glance.

To cope with this, just wrap around it with parentheses. 

				
					let f a b = a*b

					f 5 3 -- works
					f (-5) 3 -- Awesome

Python is awesome, they say.

I'm strongly eager to learn Python and I know how powerful the language is. I know I'm gonna need to implement things using this language (which I declined to learn for years). Here I'm with my background in JS and get started learning it.

Start writing a function, and call it

JS
PYTHON
I'm glad Python has a similar ability as JavaScript which everything is object, also it is a function. This makes the transition from functional JavaScript to functional Python far less painful than I thought. You can pass in functions as parameters in JavaScript and this is legal in Python too. I love it.
However, Python doesn't allow me to define a function body within the other function's argument like JavaScript's style. I need to use "lambda" expression instead to achieve exactly the same thing. Looks a bit like this pollutes the code.

Collections

JS
PYTHON
When dealing with various collections in Python, I have no argument why Python is so famous in data processing application. Many of them are so versatile! Mutating among different types of built-in collections in Python is so easy and beautiful. It supports broader range of collection functionalities than in JavaScript.

Iterate!

JS
PYTHON
I find that it's quite difficult in the first few steps to settle with the way iteration works in Python. In JavaScript, [Array] comes with a great collection of iterative functions which beautifies the code and also allows functional taste. But in Python, built-in functions hold the recipe of functional methods instead. It's a preference so I can't guarantee which is better. But, at least, I'm forced to end up use the list comprehension instead. Thus the code looks syntactically semantic, more Pythonic and clearly more readable. (Even though that's not what I expected to do)

Closure with currying

JS
PYTHON
Closures do magic in JavaScript and I believe most JavaScript developers love it. In a sense of functional programming, closures are inevitably leading casts in the storyline (to me, closures are signature of writing functional code). When learning Python, I am really happy to know that Python officially supports closures as well. The way we need to write closures in Python is kind of familiar and doesn't even need any learning curve at all for JavaScripters.
JavaScript is a very good language at closure composing and I find Python has a comparable ability. But to me, JavaScript code still looks a bit neater especially when you happen to write multiple-cascading-level nested closures. You'll be doomed by the indentation traps. Scoping the nested closures with braces also create a pyramid of doom but it's lesser evil, isn't it?
